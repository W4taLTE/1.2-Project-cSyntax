üí° Transition Fundamentals

CSS Transitions memungkinkan Anda mengubah nilai properti secara halus selama durasi tertentu, menciptakan efek animasi yang smooth.

transitions.css
/* Transition dasar */
.box {
  width: 100px;
  height: 100px;
  background: #9333ea;
  transition: all 0.3s ease;
}

.box:hover {
  background: #a855f7;
  transform: scale(1.1);
}

/* Multiple transitions */
.button {
  transition: background 0.2s ease, transform 0.4s cubic-bezier(0.175, 0.885, 0.32, 1.275);
}

üìù Penjelasan Detail:

‚Ä¢ transition-property - Properti yang akan dianimasikan
‚Ä¢ transition-duration - Durasi animasi (contoh: 0.3s)
‚Ä¢ transition-timing-function - Fungsi waktu (ease, linear, cubic-bezier)
‚Ä¢ transition-delay - Penundaan sebelum animasi dimulai
‚Ä¢ transition - Shorthand untuk semua properti di atas

üìù Penjelasan Detail:

‚Ä¢ Timing Functions - Mengontrol percepatan animasi
‚Ä¢ Durations - Waktu yang dibutuhkan animasi untuk selesai
‚Ä¢ cubic-bezier() - Memungkinkan pembuatan kurva waktu kustom
‚Ä¢ steps() - Animasi dengan langkah diskrit (bukan smooth)
‚Ä¢ Nilai durasi antara 100ms-300ms biasanya ideal untuk interaksi pengguna

üéØ Best Practices

Gunakan transitions untuk meningkatkan pengalaman pengguna dengan animasi yang halus dan performant.

best-practices.css
/* ‚úÖ Good - Performant properties */
.element {
  transition: transform 0.3s ease, opacity 0.3s ease;
}

/* ‚ùå Avoid - Heavy animations */

  transition: all 0.3s ease;
  width: 100px;
}

/* ‚úÖ Good - Will-change optimization */

  will-change: transform, opacity;
}

/* ‚ùå Avoid - Overusing will-change */
* {
  will-change: all;
}

üìù Penjelasan Detail:

‚Ä¢ Animasikan properti yang ringan - transform dan opacity paling efisien
‚Ä¢ Hindari animasi layout properties - margin, padding, width/height
‚Ä¢ Gunakan will-change dengan bijak - Hanya untuk elemen yang benar-benar berubah
‚Ä¢ Batasi jumlah animasi - Terlalu banyak animasi membingungkan pengguna
‚Ä¢ Gunakan prefers-reduced-motion - Hormati preferensi pengguna
