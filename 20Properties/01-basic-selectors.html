üí° Selector Fundamental

CSS Selectors adalah pola yang digunakan untuk memilih elemen HTML yang ingin diberi styling. Memahami selectors adalah kunci untuk menguasai CSS.

selectors.css
/* Universal Selector */
* {
  margin: 0;
  padding: 0;
}

/* Element Selector */
h1 {
  color: #9333ea;
  font-size: 2.5rem;
}

/* Class Selector */
.header {
  background: linear-gradient(45deg, #9333ea, #a855f7);
  padding: 2rem;
}

/* ID Selector */
#main-content {
  max-width: 1200px;
  margin: 0 auto;
}

üìù Penjelasan Detail:

‚Ä¢ * - Universal Selector memilih semua elemen
‚Ä¢ h1, p, div - Element Selector memilih berdasarkan tag HTML
‚Ä¢ .class-name - Class Selector dimulai dengan titik (.)
‚Ä¢ #id-name - ID Selector dimulai dengan hash (#)
‚Ä¢ h1.title - Combined Selector untuk spesifitas lebih tinggi

selector-values.css
/* Priority Table (Specificity) */
*            /* 0,0,0,0 */
li           /* 0,0,0,1 */
.class       /* 0,0,1,0 */
#id          /* 0,1,0,0 */
style=""     /* 1,0,0,0 */

/* Common Patterns */
.btn.primary    /* Multiple classes */
div.container   /* Element + class */
#header .logo  /* ID + descendant */
ul li:first-child /* Pseudo-selector */

üìù Penjelasan Detail:

‚Ä¢ Specificity menentukan prioritas selector yang akan diterapkan
‚Ä¢ Semakin tinggi nilai specificity, semakin tinggi prioritasnya
‚Ä¢ !important akan override semua specificity
‚Ä¢ Best practice: hindari penggunaan !important yang berlebihan

üí° Pro Tips

üéØ Best Practices

Gunakan class selectors untuk styling utama, hindari over-nesting (max 3 levels), dan prioritaskan readability over cleverness.

best-practices.css
/* ‚úÖ Good - Readable and maintainable */
.btn-primary {
  background: #9333ea;
  color: white;
}

.card .title {
  font-size: 1.5rem;
}

/* ‚ùå Avoid - Too specific */
div.container > section.main > article.post > h2.title {
  color: #9333ea;
}

/* ‚ùå Avoid - Over-qualified */
div#header div.container ul.nav li.item a.link {
  text-decoration: none;
}

üìù Penjelasan Detail:

‚Ä¢ Keep it simple - gunakan selector sesederhana mungkin
‚Ä¢ Avoid deep nesting - maksimal 3 level untuk maintainability
‚Ä¢ Use meaningful names - class names harus deskriptif
‚Ä¢ Follow BEM methodology - Block__Element--Modifier pattern
‚Ä¢ Performance matters - ID dan class selectors paling cepat
